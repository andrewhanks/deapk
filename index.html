What is Deapk?

Deapk is an open source tool that can directly decompile the APK files to Android project with only one step.

It is applied to Android platform, and the study and development personnel can use it. It is so simple that even the new learners can use it.

It can be used in Windows, Linux and Mac system.

Usage

  • Drag APK to Deapk.bat (Windows only)
  • Input "deapk xxx.apk" in the command line (All platform supported)

So easy, right?

Before

Before Deapk is developed, if you want to decompile an Android application program (i.e. a file with APK as the extension name, hereinafter referred to as “APK”) so as to generate a complete Eclipse project with Java source code and resource files, the following steps are usually taken:

  1. Input ApkTool d a.apk and decompile all resource files and smali files in APK.

  2. Extract the classes.dex files in APK and input Dex2Jar classes.dex to generate all Java Class files;

  3. Open JD-GUI (or input jad) so as to generate all Java source code files;

  4. Open Eclipse and create a new Android project with “package name” and “project name”, etc filled in;

  5. Import all files that shall be decompiled into this project as needed.

Generally, this requires a lot of time and efforts, and you also need to understand what shall be done for each step and how to do? This is very troublesome for new learners.

After

Now, everything has become simpler. You just need to drag APK to Deapk or input deapk xxx.apk in the command line, and then you can get the results you expect. The Eclipse project generated after de-compilation is very complete, just like our own project, so it is a necessary tool for project learning and study.

Deapk provides a solution which will integrate the current best and most mature open source decompilation tools together to make such tools play their core strength, and meanwhile make them complement each other. Its development is only for presenting you the best results.

Attentions

Just like ApkTool, Dex2Jar and other open source tools, Deapk only provides a more convenient way for your study of excellent project. If you use it for illegal purposes with any consequences resulted, the developer of Deapk will not assume any liability and so please make clear confirmation before use. Furthermore, it is suggested to well protect the codes of the commercial Android project with Proguard or other excellent tools.

In addition, although Android application is easily to be decompiled, no one can restore the non-open source project to source codes at present no matter what technology is used and how simple the method is.
